# 文献评审员 (Literature Reviewer)

## 触发条件
用户提供以下任意一种输入时启动：
- 本地 PDF 路径（如 `/path/to/paper.pdf`）
- arXiv ID（如 `2307.12585`）
- 论文 URL

## 工作流

### Step 1：获取文献

**本地 PDF**：
```
Read(file_path)
```

**arXiv ID**：先用 `mcp__arxiv__get_abstract` 获取摘要确认相关性，再用 `mcp__arxiv__download_paper` 下载全文。

**URL**：用 `WebFetch` 获取内容。

### Step 2：结构化分析

阅读全文后，按以下维度分析：

1. **核心贡献**：论文解决了什么问题？主要声明是什么？
2. **方法论**：关键方法、实验设计、数学框架
3. **关键结果**：主要数据、图表结论、定量指标
4. **创新点**：与已有工作的区别，引用对比的基准
5. **局限性**：作者自述的局限 + 评审者发现的潜在问题
6. **批判性评估**：逻辑漏洞、未验证假设、可重复性问题

### Step 3：生成报告

将分析整合为 Markdown 报告，保存到 `reviews/<作者>_<年份>_<关键词>.md`。

## 报告模板

```markdown
# 文献评审：[标题]

**arXiv/DOI**: [ID]  
**作者**: [作者列表]  
**发表时间**: [日期]  
**评审日期**: [今日日期]

---

## 核心贡献

[1-2 段，明确论文解决的问题和主要声明]

## 方法论

[详细描述关键方法，包括公式、实验设计、数据集]

## 关键结果

[引用具体数字、图表编号、定量比较]

## 创新点

- [创新点 1]
- [创新点 2]

## 局限性

- [局限性 1（作者自述）]
- [局限性 2（评审者发现）]

## 批判性评估

[逻辑漏洞、假设合理性、与领域已知结论的一致性]

## 与已有工作的关系

[引用对比的关键文献，说明本文在领域中的位置]

## 总结评分（可选）

| 维度 | 评分（1-5） | 备注 |
|------|------------|------|
| 新颖性 | | |
| 方法严谨性 | | |
| 结果可信度 | | |
| 写作清晰度 | | |
```

## 注意事项

- 引用具体图表编号和数字，不做空泛描述
- 区分"作者声明"和"评审者判断"
- 若文献与已有笔记（`notes/`、`skills/`）有关联，指出一致性或矛盾
- 报告保存后，在 `papers/timeline.md` 中检查是否需要添加该文献条目
